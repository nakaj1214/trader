# 実装者サブエージェント プロンプトテンプレート

実装者サブエージェントをディスパッチする際にこのテンプレートを使用します。

```
Task tool (general-purpose):
  description: "タスク N を実装: [タスク名]"
  prompt: |
    あなたはタスク N: [タスク名] を実装します

    ## タスクの説明

    [計画からのタスクの全文をここに貼り付ける - サブエージェントにファイルを読ませない]

    ## コンテキスト

    [背景説明: これがどこに位置するか、依存関係、アーキテクチャのコンテキスト]

    ## 開始前に

    以下について質問がある場合:
    - 要件や受け入れ基準
    - アプローチや実装戦略
    - 依存関係や前提条件
    - タスクの説明で不明な点

    **今すぐ質問してください。** 作業を開始する前に懸念事項を提起してください。

    ## あなたの役割

    要件が明確になったら:
    1. タスクが指定する内容を正確に実装する
    2. テストを書く（タスクが指示する場合は TDD に従う）
    3. 実装が動作することを確認する
    4. 作業をコミットする
    5. セルフレビューを行う（以下参照）
    6. 報告する

    作業ディレクトリ: [directory]

    **作業中に:** 予期しないことや不明なことに遭遇した場合、**質問してください。**
    一旦立ち止まって確認することはいつでも OK です。推測や仮定はしないでください。

    ## 報告前に: セルフレビュー

    新鮮な目で自分の作業をレビューしてください。以下を自問してください:

    **完全性:**
    - 仕様のすべてを完全に実装したか？
    - 見落とした要件はないか？
    - 処理していないエッジケースはないか？

    **品質:**
    - これは自分のベストワークか？
    - 名前は明確で正確か（動作方法ではなく、何をするかに合致しているか）？
    - コードはクリーンで保守可能か？

    **規律:**
    - 過剰構築を避けたか（YAGNI）？
    - 要求されたものだけを構築したか？
    - コードベースの既存パターンに従ったか？

    **テスト:**
    - テストは実際の動作を検証しているか（モックの動作だけでなく）？
    - 必要に応じて TDD に従ったか？
    - テストは包括的か？

    セルフレビューで問題が見つかった場合は、報告前に今修正してください。

    ## レポートフォーマット

    完了後、以下を報告:
    - 実装した内容
    - テストした内容とテスト結果
    - 変更したファイル
    - セルフレビューの所見（ある場合）
    - 問題点や懸念事項
```
