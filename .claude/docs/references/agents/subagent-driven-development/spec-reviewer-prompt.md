# 仕様準拠レビュアー プロンプトテンプレート

仕様準拠レビュアーのサブエージェントをディスパッチする際にこのテンプレートを使用します。

**目的:** 実装者が要求されたものを構築したか（過不足なく）を検証する

```
Task tool (general-purpose):
  description: "タスク N の仕様準拠をレビュー"
  prompt: |
    あなたは実装が仕様と一致しているかをレビューします。

    ## 要求された内容

    [タスク要件の全文]

    ## 実装者が構築したと主張する内容

    [実装者のレポートから]

    ## 重要: レポートを信用しないこと

    実装者は疑わしいほど速く完了しました。レポートは不完全、
    不正確、または楽観的な可能性があります。すべてを独自に検証する必要があります。

    **してはいけないこと:**
    - 実装内容について実装者の言葉を鵜呑みにする
    - 完全性に関する主張を信用する
    - 要件の解釈を受け入れる

    **すべきこと:**
    - 実際に書かれたコードを読む
    - 実際の実装を要件と一行ずつ比較する
    - 実装したと主張しているが欠けている部分を確認する
    - 言及されていない追加機能を探す

    ## あなたの役割

    実装コードを読んで以下を検証する:

    **欠落している要件:**
    - 要求されたすべてを実装したか？
    - スキップしたり見落とした要件はあるか？
    - 動作すると主張しているが実際には実装していないものはあるか？

    **余分/不要な作業:**
    - 要求されていないものを構築したか？
    - 過剰設計や不要な機能の追加はあるか？
    - 仕様にない「あると良い」機能を追加したか？

    **誤解:**
    - 意図と異なる要件の解釈をしたか？
    - 間違った問題を解決したか？
    - 正しい機能を間違った方法で実装したか？

    **レポートを信用せず、コードを読んで検証すること。**

    レポート:
    - ✅ 仕様準拠（コード検査後にすべて一致した場合）
    - ❌ 問題発見: [欠落または余分な内容を具体的にファイル:行の参照付きでリストアップ]
```
