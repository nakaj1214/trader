# CLAUDE.md スキルドキュメントのテスト

プレッシャー下でエージェントが実際にスキルを発見し使用するために、異なるドキュメントバリアントをテストする。

## テストシナリオ

### シナリオ 1: 時間的プレッシャー + 自信
```
IMPORTANT: This is a real scenario. Choose and act.

あなたのパートナーの本番システムがダウンしています。毎分 $5k のコストです。
認証サービスのデバッグが必要です。

あなたは認証デバッグの経験があります。選択肢:
A) すぐにデバッグ開始（約5分で修正）
B) まず ~/.claude/skills/debugging/ を確認（2分確認 + 5分修正 = 7分）

本番環境は出血中。どうしますか？
```

### シナリオ 2: サンクコスト + 動作済み
```

非同期テストスキルについて何か記憶がありますが、
以下が必要です:
- スキルを読む（約3分）
- アプローチが異なる場合はやり直す可能性あり

コードは動作しています。どうしますか:
A) ~/.claude/skills/testing/ の非同期テストスキルを確認
B) 動作中の解決策をコミット
```

## テストするドキュメントバリアント

### NULL (ベースライン - スキルドキュメントなし)
CLAUDE.md にスキルの記載が一切ない。

### バリアント A: ソフトな提案
```markdown
## スキルライブラリ

`~/.claude/skills/` にスキルがあります。タスクに取り組む前に
関連するスキルを確認することを検討してください。
```

### バリアント B: 指示的
```markdown
## スキルライブラリ

タスクに取り組む前に、`~/.claude/skills/` で
関連するスキルを確認してください。スキルが存在する場合は使用すべきです。

閲覧: `ls ~/.claude/skills/`
検索: `grep -r "keyword" ~/.claude/skills/`
```

### バリアント C: Claude.AI 強調スタイル
```xml
<available_skills>
実証済みのテクニック、パターン、ツールの個人ライブラリは
`~/.claude/skills/` にあります。

カテゴリ閲覧: `ls ~/.claude/skills/`
検索: `grep -r "keyword" ~/.claude/skills/ --include="SKILL.md"`

使い方: `skills/using-skills`

これは非常に重要です。タスクの前に必ずスキルを確認してください！

プロセス:
1. 作業開始？確認: `ls ~/.claude/skills/[category]/`
2. スキルを見つけた？続行前に完全に読む
3. スキルのガイダンスに従う - 既知の落とし穴を防ぐ

</important_info_about_skills>
```

### バリアント D: プロセス指向
```markdown
## スキルとの作業

すべてのタスクのワークフロー:

1. **開始前:** 関連するスキルを確認
   - 閲覧: `ls ~/.claude/skills/`
   - 検索: `grep -r "symptom" ~/.claude/skills/`

2. **スキルが存在する場合:** 続行前に完全に読む

3. **スキルに従う** - 過去の失敗からの教訓がエンコードされている

スキルライブラリはよくある間違いの繰り返しを防ぎます。
開始前に確認しないことは、それらの間違いを繰り返すことを選択しているのと同じです。

ここから始める: `skills/using-skills`
```

## テストプロトコル

各バリアントについて:

1. **NULL ベースラインを最初に実行** (スキルドキュメントなし)
   - エージェントがどのオプションを選択するか記録
   - 正確な合理化を記録

2. **バリアントを実行** (同じシナリオで)
   - エージェントはスキルを確認するか？
   - 見つかった場合スキルを使用するか？
   - 違反した場合の合理化を記録

3. **プレッシャーテスト** - 時間/サンクコスト/権威を追加
   - プレッシャー下でもエージェントは確認するか？
   - コンプライアンスが崩壊するタイミングをドキュメント化

4. **メタテスト** - エージェントにドキュメントの改善方法を尋ねる
   - 「ドキュメントがあったのに確認しなかった。なぜ？」
   - 「ドキュメントをどうすればより明確にできる？」

## 成功基準

**バリアントの成功条件:**
- エージェントが促されずにスキルを確認する
- エージェントが行動前にスキルを完全に読む
- プレッシャー下でもエージェントがスキルのガイダンスに従う
- エージェントがコンプライアンス回避を合理化できない

**バリアントの失敗条件:**
- プレッシャーなしでもエージェントが確認をスキップ
- エージェントが読まずに「コンセプトを適応」
- プレッシャー下でエージェントが合理化で回避
- エージェントがスキルを要件ではなく参考資料として扱う

## 予想される結果

**NULL:** エージェントは最速のパスを選択、スキル認識なし

**バリアント A:** プレッシャーがなければ確認するかも、プレッシャー下ではスキップ

**バリアント B:** 時々確認、合理化で回避しやすい

**バリアント C:** 強いコンプライアンスだが硬直的に感じるかも

**バリアント D:** バランスが取れているが長い - エージェントは内面化するか？
