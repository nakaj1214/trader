# プラグイン構造スキル

Claude Code プラグインのアーキテクチャ、ディレクトリレイアウト、ベストプラクティスに関する包括的なガイダンスです。

## 概要

このスキルは以下に関する詳細な知識を提供します:
- プラグインのディレクトリ構造と組織化
- `plugin.json` マニフェスト設定
- コンポーネントの組織化（コマンド、エージェント、スキル、フック）
- 自動検出メカニズム
- `${CLAUDE_PLUGIN_ROOT}` によるポータブルなパス参照
- ファイル命名規則

## スキル構造

### SKILL.md（1,619語）

以下をカバーするコアスキルコンテンツ:
- ディレクトリ構造の概要
- プラグインマニフェスト（plugin.json）フィールド
- コンポーネント組織化パターン
- ${CLAUDE_PLUGIN_ROOT} の使い方
- ファイル命名規則
- 自動検出メカニズム
- ベストプラクティス
- よくあるパターン
- トラブルシューティング

### リファレンス

深掘りのための詳細ドキュメント:

- **manifest-reference.md**: 完全な `plugin.json` フィールドリファレンス
  - すべてのフィールドの説明と例
  - パス解決ルール
  - バリデーションガイドライン
  - 最小限 vs. 完全なマニフェストの例

- **component-patterns.md**: 高度な組織化パターン
  - コンポーネントのライフサイクル（検出、有効化）
  - コマンド組織化パターン
  - エージェント組織化パターン
  - スキル組織化パターン
  - フック組織化パターン
  - スクリプト組織化パターン
  - クロスコンポーネントパターン
  - スケーラビリティのベストプラクティス

### 例

3つの完全なプラグイン例:

- **minimal-plugin.md**: 最もシンプルなプラグイン
  - 単一コマンド
  - 最小限のマニフェスト
  - このパターンを使用する場面

- **standard-plugin.md**: 適切に構造化された本番プラグイン
  - 複数のコンポーネント（コマンド、エージェント、スキル、フック）
  - メタデータ付きの完全なマニフェスト
  - 充実したスキル構造
  - コンポーネント間の統合

- **advanced-plugin.md**: エンタープライズグレードのプラグイン
  - 多層的な組織化
  - MCP サーバー統合
  - 共有ライブラリ
  - 設定管理
  - セキュリティ自動化
  - モニタリング統合

## このスキルがトリガーされる場面

Claude Code は以下の場合にこのスキルを有効にします:
- 「プラグインを作成」「プラグインのスキャフォールディング」を依頼された場合
- 「プラグイン構造の理解」が必要な場合
- 「プラグインコンポーネントの整理」を求められた場合
- 「plugin.json のセットアップ」が必要な場合
- 「${CLAUDE_PLUGIN_ROOT}」の使い方について質問された場合
- 「コマンド/エージェント/スキル/フックの追加」を求められた場合
- 「自動検出の設定」のヘルプが必要な場合
- プラグインアーキテクチャやベストプラクティスについて質問された場合

## 段階的な情報開示

このスキルはコンテキストを管理するために段階的な情報開示を使用します:

1. **SKILL.md**（約1600語）: コアコンセプトとワークフロー
2. **リファレンス**（約6000語）: 詳細なフィールドリファレンスとパターン
3. **例**（約8000語）: 完全な動作例

Claude はタスクに応じて必要な場合のみリファレンスと例を読み込みます。

## 関連スキル

このスキルは以下と組み合わせて使用できます:
- **hook-development**: プラグインフックの作成
- **mcp-integration**: MCP サーバーの統合（利用可能な場合）
- **marketplace-publishing**: プラグインの公開（利用可能な場合）

## メンテナンス

このスキルを更新するには:
1. SKILL.md をコアコンセプトに焦点を当てて簡潔に保つ
2. 詳細情報を references/ に移動する
3. よくあるパターンの新しい examples/ を追加する
4. SKILL.md フロントマターのバージョンを更新する
5. すべてのドキュメントで命令形/不定詞形を使用する
