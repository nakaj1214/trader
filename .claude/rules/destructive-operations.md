# 破壊的操作の制限

データ損失を防ぐための安全ルール。すべてのプロジェクトに適用。

## ファイル削除

- ファイル削除時は `archive/` フォルダにコピーしてから削除する
- `archive/` が存在しない場合はプロジェクトルートに作成する
- `rm -rf` の使用は禁止（`rm` 単体も確認を取ってから実行）

## コード削除

- コードの削除はコメントアウトで行う（完全削除は後のセッションで確認後）
- 大量のコード削除が必要な場合はユーザーに確認を取る
- 未使用コードの判断は呼び出し元を必ず確認してから

## Git 操作

- `git reset --hard` は禁止
- `git clean -f` は禁止
- `git push --force` は禁止（ユーザーが明示的に要求した場合のみ）
- `git branch -D` は禁止（`-d` を使用すること）

## データベース操作

- `DROP TABLE` / `DROP DATABASE` は禁止
- `TRUNCATE TABLE` はユーザー確認必須
- マイグレーションのロールバックはバックアップ確認後

## 例外

- テスト用の一時ファイル（`/tmp/`, `__pycache__/`, `.pytest_cache/`）は削除可
- ビルド成果物（`dist/`, `build/`, `node_modules/`）はユーザー確認後に削除可
