# エラー報告の標準形式

エラーを `docs/error/error.md` に記載する際の統一形式。
構造化された報告により、原因特定と修正の速度を向上させる。

## 記載テンプレート

```markdown
## エラー: {短い説明}

- **発生場所**: {ファイルパス、URL、コマンド}
- **エラーメッセージ**:
  ```
  {完全なエラー出力を貼り付け}
  ```
- **再現手順**:
  1. {何をしたらエラーが出たか}
  2. {ステップ2}
- **環境**: {ローカル / CI / デプロイ先}
- **発生頻度**: {毎回 / 時々 / 初回のみ}
```

## ルール

- エラーメッセージは省略せず完全に貼り付ける
- スタックトレースがある場合は全文を含める
- スクリーンショットがある場合はパスを記載する
- 複数のエラーがある場合は1エラー1セクションで分離する
- 修正済みのエラーは `<!-- -->` でコメントアウトする

## error.md の読み取り側

Claude がエラードキュメントを読む際:
- 非コメント行のエラーのみを対象とする
- 発生場所から該当ファイルを特定する
- エラーメッセージから原因を推定する
- 再現手順に従って状況を再現する
