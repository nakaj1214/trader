# 計画の実行

## 概要

計画を読み込み、批判的にレビューし、全タスクを連続で実行し、完了後にまとめて報告する。

**核心原則:** ブロッカーが発生しない限り、全タスクを中断なしで連続実行する。

**開始時に宣言:** 「implement-plansスキルを使用してこの計画を実装します。」

## プロセス

### Step 1: 計画を読み込んでレビューする
1. 計画ファイルを読む
2. 批判的にレビュー - 計画について質問や懸念を特定する
3. 懸念がある場合: 開始前にユーザーに提起する
4. 懸念がない場合: TodoWriteを作成して進む

### Step 2: 全タスクを連続実行する
**中断なしで全タスクを実行する。**

各タスクについて:
1. in_progressとしてマーク
2. 各ステップを正確に従う（計画は小さなステップに分かれている）
3. 指定された検証を実行する
4. completedとしてマーク
5. **次のタスクへ即座に進む**（ユーザーのフィードバックを待たない）

### Step 3: 完了報告
全タスク完了後にまとめて報告する:
- 実装した内容の一覧
- 各タスクの検証結果
- 発見した問題や注意点
- 「全タスクが完了しました。確認をお願いします。」と言う

### Step 4: フィードバック対応
ユーザーからのフィードバックに基づいて:
- 必要に応じて修正を適用する

## 停止してヘルプを求める場合

**以下の場合のみ実行を停止する:**
- 解決不能なブロッカーに当たった（依存関係の不足、テスト失敗、指示が不明確）
- 計画に重大なギャップがある
- 指示を理解できない
- 検証が繰り返し失敗する

**推測するのではなく明確化を求める。**
**ただし、自力で解決できる問題は停止せずに解決して進む。**

## 覚えておくこと
- まず計画を批判的にレビューする
- 計画のステップを正確に従う
- 検証をスキップしない
- **途中で停止しない** — ブロッカーがない限り全タスクを連続実行する
- ブロックされたら停止し、推測しない
- 明示的なユーザーの同意なしにmain/masterブランチで実装を開始しない
