# refactor-with-read-first

同一ファイルへの細粒度 Edit の繰り返しを検出し、全体を Read → 設計固め → 一括 Write に切り替える。

## トリガー条件

同一ファイルへの Edit が5回を超えたとき（edit-tracker hook の警告と連動）。

## 実行手順

### 1. 現状の確認

対象ファイルを Read で全体読み込みする。

### 2. 変更意図の整理

これまでの Edit 履歴から、何を達成しようとしているかを整理する:
- 追加した機能・修正
- まだ未完了の変更
- 矛盾する変更（追加→削除→再追加など）

### 3. 完成形の設計

ファイル全体の完成形を設計する:
- 既存コードのうち変更不要な部分を特定
- 変更が必要な部分を一括でまとめる
- 構文チェック（`node --check` / `php -l`）が通る形にする

### 4. 一括 Write

設計した完成形で Write ツールを使い、ファイル全体を一括書き換える。

### 5. 構文検証

```bash
# JS の場合
node --check {file}

# PHP の場合
docker compose exec laravel php -l {file}
```

## 注意事項

- Write 前に必ず Read で最新の内容を確認する
- 変更箇所が少ない場合（1-2箇所）は Edit のままで良い
- このスキルは「5回超の Edit 繰り返し」を防ぐためのもので、通常の Edit を禁止するものではない
