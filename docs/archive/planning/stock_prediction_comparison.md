# 株価予測の3つのアプローチ比較（1週間予測システム）

## 概要
1週間の株価上昇を予測する3つのアプローチを、初心者向けの観点から比較します。

---

## 1. OpenAI API (GPT-4o) - LLMベースアプローチ

### 予測精度（短期：1週間）
**評価: ⭐⭐⭐☆☆ (3/5)**
- **長所**:
  - テクニカル指標とニュースを自然言語で入力できる
  - 複雑なパターンや市場心理を理解できる可能性
  - ファイナンシャルニュースの文脈を解釈できる

- **短所**:
  - 1週間という短期での予測には向かない
  - 学習データのカットオフ（2024年4月まで）により最新情報がない
  - 確率的出力（非決定的）で再現性が低い
  - 実際の市場変動の予測には効果が限定的

### セットアップの複雑さ
**評価: ⭐⭐⭐⭐☆ (4/5 - 簡単)**
- OpenAI APIキーを取得するだけ
- Pythonで数行のコードで実装可能
- APIの使用方法がドキュメント化されている
- 環境構築が最小限

### 実行コスト
**評価: ⭐⭐☆☆☆ (2/5 - 高い)**
- API呼び出し: $0.005-$0.015 / 1K tokens（入力）
- 1回の予測で複数銘柄・複数指標を処理すると、$1-$5/日の費用
- 月額: $30-$150（毎日実行の場合）
- スケーリングが困難（銘柄数が増えると指数関数的にコスト増）

### データ要件
**評価: ⭐⭐⭐⭐⭐ (5/5 - 最小限)**
- テクニカル指標（移動平均線、RSI、MACD等）
- 直近のニュースヘッドライン
- 過去1-2週間の価格データのみ

### 初心者向けの評価

**メリット**:
1. 技術的な知識がほぼ不要
2. ニュースと指標を日本語で説明できれば機能
3. 実装が迅速（1時間以内）
4. プロンプトエンジニアリングで結果を調整可能

**デメリット**:
1. **継続的なコスト負担** - 本格運用には経費がネック
2. 株式市場のデータサイエンス知識がなくても使えるが、出力の妥当性を判断できない
3. APIの利用規約変更のリスク
4. 短期予測には原理的に不適切（LLMは時系列予測に最適化されていない）
5. バックテストができない（過去データで検証不可）

---

## 2. scikit-learn - 従来的な機械学習

### 予測精度（短期：1週間）
**評価: ⭐⭐⭐⭐☆ (4/5)**
- **長所**:
  - 正しく構築すれば、中程度の精度を達成可能
  - ハイパーパラメータチューニングで最適化できる
  - クロスバリデーションで性能を正確に評価
  - バックテストが可能

- **短所**:
  - 特徴エンジニアリングが非常に重要（初心者には困難）
  - 過学習のリスク
  - 1週間という短期では、ランダムノイズに埋もれることが多い
  - 市場データは非定常で、モデルの再学習が必須

### セットアップの複雑さ
**評価: ⭐⭐⭐☆☆ (3/5 - 中程度)**
```python
# インストール
pip install scikit-learn pandas numpy

# 基本的な流れ
1. データ取得（yfinance等）
2. テクニカル指標の計算
3. データ分割（訓練/テスト）
4. モデル訓練
5. 予測
```
- セットアップ自体は簡単だが、「効果的に使う」には機械学習の知識が必要
- 初心者は「どの特徴を選ぶか」で迷う

### 実行コスト
**評価: ⭐⭐⭐⭐⭐ (5/5 - 無料)**
- ライブラリ: 完全無料・オープンソース
- サーバー: 自分のPC/サーバーで実行
- スケーリング: 費用増なし（処理時間は増加）

### データ要件
**評価: ⭐⭐⭐☆☆ (3/5)**
- テクニカル指標の計算に過去60-250日のデータが必要
- データの品質：高い（ノイズに強いモデルが多い）
- 複数の銘柄で訓練する場合、数千～数万サンプルが理想

### 初心者向けの評価

**メリット**:
1. 継続的コストがゼロ
2. オープンソースで完全にカスタマイズ可能
3. 機械学習の基礎を学べる
4. バックテストで過去のパフォーマンスを検証
5. スケーリングが容易（多くの銘柄に対応可能）

**デメリット**:
1. **学習曲線が急** - 機械学習の基本知識が必須
2. 特徴エンジニアリングが複雑（初心者は何を入力すべきか不明）
3. 過学習との闘い（訓練精度は高いが、実運用で失敗する）
4. データの前処理に時間がかかる
5. 「なぜそう予測したか」の説明性が低い（ブラックボックス）
6. モデルの再訓練が定期的に必要（月1回～週1回）

## 3. Prophet (Meta) - 時系列予測

### 予測精度（短期：1週間）
**評価: ⭐⭐⭐⭐☆ (4/5)**
- **長所**:
  - 時系列データに特化
  - トレンドと季節性を自動認識
  - 外部変数（ニュース、経済指標）を追加可能
  - 1週間程度の短期予測に最適設計

- **短所**:
  - 株価は「季節性」が弱く、Prophetの強みを活かしにくい
  - 急激な変動（ギャップ）に対応困難
  - VIXが高い（ボラティリティが高い）時期は精度低下

### セットアップの複雑さ
**評価: ⭐⭐⭐⭐☆ (4/5 - 簡単)**
```python
# インストール
pip install pystan prophet

# 基本的な流れ
from prophet import Prophet

model = Prophet()
model.fit(df)
future = model.make_future_dataframe(periods=7)
forecast = model.predict(future)
```
- コード量が少ない（10-20行で動作）
- ハイパーパラメータが少ない（デフォルト設定で多くの場合に機能）
- scikit-learnより習得が簡単

### 実行コスト
**評価: ⭐⭐⭐⭐⭐ (5/5 - 無料)**
- ライブラリ: 完全無料・オープンソース
- サーバー: 自分のPC/サーバーで実行
- スケーリング: 費用増なし

### データ要件
**評価: ⭐⭐⭐⭐☆ (4/5)**
- 最小要件: 過去60日程度のデータ
- 理想的: 過去1-2年のデータ（季節性を学ぶため）
- 形式: シンプル（datetime, value の2列）
- 欠損値への耐性が高い

### 初心者向けの評価

**メリット**:
1. **最も簡単に始められる**
2. 完全無料
3. ドキュメントと例が豊富
4. チューニングの余地は少なく、デフォルト設定で動作
5. 予測区間を自動計算（信頼区間が表示される）
6. Facebookが開発・メンテナンス（信頼性高い）

**デメリット**:
1. 株価特有の問題に弱い（季節性なし、ノイズが多い）
2. サポートベクトル等の機械学習概念は学べない
3. 精度が「中程度」レベルに留まる傾向
4. 外部変数（ニュース）の統合が手作業
5. 急激な相場変動を予測できない

## 初心者への推奨事項

### 🎯 **推奨アプローチ: Prophet（段階的アプローチ）**

#### フェーズ1：Prophet で始める（最初の2-3週間）
**理由**:
1. 最も簡単に始められる
2. コストゼロで実験可能
3. 1-2週間で動作するシステム構築可能
4. バックテストで自分の予測精度を確認できる

**実装例**:
```python
import pandas as pd
from prophet import Prophet
import yfinance as yf

# データ取得
df = yf.download('AAPL', start='2023-01-01')
df = df.reset_index()[['Date', 'Close']]
df.columns = ['ds', 'y']

# モデル作成
model = Prophet()
model.fit(df)

# 1週間先予測
future = model.make_future_dataframe(periods=7)
forecast = model.predict(future)

print(forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']].tail(7))
```

**達成できること**:
- 自分で予測を実行できる
- 精度を測定できる
- 基本的な時系列の概念を理解できる

## 初心者が陥りやすい落とし穴

### 1. **精度への過度な期待**
- 株価予測は原理的に難しい
- 50-55%の的中率でも、費用対効果があれば成功
- 完全予測は不可能（市場効率仮説）

### 2. **バックテスト偏差（Backtest Bias）**
- 過去データで高精度でも、実運用では失敗することが多い
- 理由: 市場環境の変化、過学習
- 対策: 常に新しいデータで再訓練

### 3. **データ漏洩（Look-ahead Bias）**
- 訓練時に未来のデータを使ってしまう
- 特に初心者がやりやすいミス

### 4. **特徴エンジニアリングの重要性を過小評価**
- scikit-learn で失敗する理由の80%は「特徴選択」の失敗
- モデルより特徴が重要（「ガベージイン、ガベージアウト」）

```
  └─ 目標: Prophet + scikit-learn の組み合わせ

  └─ 目標: リアルマネーでの運用、定期メンテナンス
```

## 最終推奨: 「Prophet から始めて、必要に応じて高度化」

**初心者にとって最適な選択肢**:
1. **今すぐ始めたい** → **Prophet**
2. **コストが心配** → **Prophet または scikit-learn**
3. **ニュースの文脈を含めたい** → **Prophet + OpenAI API の補完**
4. **本格的な精度が必要** → **scikit-learn + アンサンブル**

Prophet は**「失敗が少なく、学習効率が高い」**最適な出発点です。
