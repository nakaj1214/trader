# USER GUIDE

対象: このシステムの通知とダッシュボードを利用するユーザー

## 1. できること

- 毎週の上昇予測銘柄を Slack で受け取る
- 必要に応じて LINE で「Slack確認リマインド」を受け取る
- Web画面で予測履歴と的中率を確認する
- 各銘柄のリスク指標・選出理由・エビデンスを確認する
- 各銘柄の推奨ポジションサイズと損切りめやすを確認する
- 予測精度の誤差分析と戦略別パフォーマンス比較を確認する
- バックテスト品質の開示情報（過剰適合リスク指標）を確認する
- マクロ環境（FF金利・イールドカーブ・VIX）をバナーで確認する
- 過去の予測に基づいた仮想投資のシミュレーションを行う

注記:
- 本システムは投資判断の補助です。自動売買は行いません。

## 2. 毎週の使い方

1. Slack 通知を開く
2. `銘柄 / 現在価格 / 予測価格 / 予測上昇率 / 信頼区間` を確認する
3. 前週の的中率と累計的中率を確認する
4. 必要に応じて証券会社画面で実際の価格を再確認する
5. 詳細を見たい場合はダッシュボードを開く

## 3. 通知の見方

- `AAPL: $250.00 → $265.00`
  - 現在価格から予測価格への見立て
- `予測 +6.0%`
  - 予測時点での上昇見込み
- `信頼区間 ±3.2%`
  - 予測の不確実性の目安
- `的中: 6/8 (75.0%)`
  - 前週予測の結果

LINE通知について:
- LINEは補助通知です（Slack確認の促進）
- LINE失敗時でも Slack が届いていれば運用は継続されます

## 4. ダッシュボードの見方

### 主なページ

- `index.html`: 最新週の予測一覧 + マクロバナー + 累計的中率
- `accuracy.html`: 週次的中率と累計推移、銘柄別ランキング、予測誤差分析、バックテスト品質開示、戦略比較
- `stock.html?ticker=...`: 銘柄ごとの予測履歴と実績推移・ポジションサイジング
- `simulator.html`: 投資シミュレータ

### index.html（サマリー）の見方

#### マクロ環境バナー

ページ上部に表示されます（`FRED_API_KEY` が設定されている場合）:

- `FF金利: 5.33%`: 現在のフェデラルファンド金利
- `10Y-2Y: +0.10%`: 10年債と2年債の利回りスプレッド（イールドカーブ）
- `VIX: 18.0`: 恐怖指数（株式市場のボラティリティ期待値）

リスクオフ判定:
- VIX > 25 または 10Y-2Y < 0（逆イールド）の場合、バナーがオレンジ系に変わります
- 理由が注記として表示されます（例: `リスクオフ: VIX=30.0 > 25.0`）
- `FRED_API_KEY` が未設定の場合、バナーは表示されません

#### 予測カード

各カードに以下の情報が表示されます:

- 銘柄名、現在価格 → 予測価格、予測上昇率
- 実績価格と的中/外れバッジ
- リスク行: `ボラ30% | β1.20 | DD-15%`
  - ボラ = 年率換算ボラティリティ（値動きの大きさ）
  - β = S&P 500 に対する感応度（1.0超で市場より大きく動く）
  - DD = 過去1年の最大下落幅
- イベントバッジ: 決算発表や配当落ち日が近い場合に表示

#### 予測ガードレールバッジ

予測上昇率が異常に大きい場合、銘柄名横にバッジが表示されます:

- `WARN`: 予測上昇率が警告しきい値（±20%）を超えています。参考程度にご確認ください。
- `CLIPPED`: 予測上昇率が上限（±30%）を超えたため、表示値をクリップしています。
  - 表示値の横に「(元値: +XX%)」と実際の予測値が付記されます。

### stock.html（銘柄詳細）の見方

銘柄名をクリックすると詳細ページに遷移します。最新週の予測に対して以下のパネルが表示されます:

- 予測ガードレール警告バナー: CLIPPED/WARN の場合、ページ上部に詳細を表示
- リスク指標パネル: ボラティリティ(20日/60日)、β値、最大DD、直近イベント
- エビデンスパネル: モメンタム・バリュー・クオリティ・低リスクの各z-scoreと総合スコア
  - z-score はスクリーニング対象銘柄内での相対位置を示します（高いほど優位）
- 選出理由パネル: スコアに寄与した上位要因（例: 1ヶ月騰落率、MACD、RSI）
- ポジションサイジングパネル: 推奨保有比率と損切りめやす

#### ポジションサイジングパネルの見方

目標ボラティリティ（年率10%）を基準に算出した推奨ポジションサイズです:

- `推奨ウェイト`: ポートフォリオに対する推奨保有比率（例: 15.2%）
  - 計算式: `目標年率ボラ ÷ 銘柄年率ボラ`（上限: max_weight_cap）
  - ボラが高いほど小さくなります
- `損切りめやす`: 現在価格からこの幅を超えて下落したら損切りを検討する水準
  - 計算式: `stop_loss_multiplier × 月次換算ボラ`（マイナス表記）
  - 例: `-6.9%` → 現在価格から 6.9% 下落で損切り検討

注意: これは機械的な計算値です。実際の投資判断では個別の状況を考慮してください。

### accuracy.html（的中率）の見方

- 累計的中率: 全期間の的中数 / 全予測数
- 週次的中率推移: 棒グラフ（週次）+ 折れ線（累計推移）
- 予測誤差分析: 予測上昇率の帯（0-5%, 5-10% 等）ごとに実績平均と比較
  - MAE（平均予測誤差）が小さいほど予測精度が高い
- 確率校正（キャリブレーション）:
  - 「予測上昇率 X%」と「実際に上昇した割合」が一致しているかを可視化します
  - Brier スコアは低いほど良好、ECE は 0 に近いほど校正精度が高い
  - データが30件以上蓄積されると表示されます
- バックテスト品質開示:
  - 過剰適合リスクを示す指標を公開しています
  - `ステータス: 試行数不足` — 評価に必要な戦略数が不足しています
  - `ステータス: 算出済み` — pbo / Reality Check p値が計算されています
  - Reality Check p値: 低いほど偶然でない可能性が高い（0.05未満が目安）
  - PBO（Probability of Backtest Overfitting）: 高いほど過剰適合リスクが大きい
  - Deflated Sharpe: 試行数ペナルティ調整済みシャープレシオの推定値
  - ※ 現在の値は簡略推定値です。参考程度にご覧ください
- 戦略別パフォーマンス比較:
  - AI予測戦略・12-1モメンタム戦略・SPYベンチマークの累積リターンをグラフと表で比較
  - 表示指標: CAGR（年率リターン）、最大ドローダウン、シャープレシオ
  - ※ 取引コスト・スリッページは考慮していません。過去の成績は将来を保証しません
  - 確定済みの予測データが蓄積されると自動で表示されます
- 銘柄別的中率ランキング: 銘柄ごとの的中率を降順で表示

### simulator.html（シミュレータ）の見方

過去の予測に従って毎週等金額で全銘柄に投資した場合の仮想損益を計算します。

1. 投資金額（$）を入力して「シミュレーション実行」をクリック
2. 結果として以下が表示されます:
   - 投資金額・累計損益・累計損益率・対象期間（週数）のサマリー
   - 週別損益推移グラフ（棒グラフ=週次損益、折れ線=累積損益）
   - 銘柄別損益テーブル（損益の大きい順）
3. 計算対象は実績価格が確定したレコードのみです

注意:
- これは過去データに基づく仮想シミュレーションです
- 手数料・税金・スリッページ・為替コストは含まれていません

## 5. よくあるケース

1. 「今週の上昇予測銘柄がありません」と出る
   - 条件に合う銘柄がなかった状態です（異常ではありません）

2. 的中率が表示されない
   - 初期運用では比較対象の実績がまだ揃っていない可能性があります

3. ダッシュボードでデータが表示されない
   - `dashboard/data/*.json` が未生成の可能性があります
   - 開発者向け確認手順: `docs/guide/gui_verification_steps.md`

4. リスクやエビデンスのパネルが表示されない
   - 最新週の予測データに補強情報が含まれていない場合は非表示になります
   - 株価データの取得に失敗した場合も自動的にスキップされます（予測自体には影響しません）

5. 予測誤差分析・戦略比較が表示されない
   - 確定済み（的中/外れ判定済み）の予測が一定数蓄積されると自動で表示されます

6. WARN/CLIPPED バッジが表示される
   - モデルが異常に大きな予測値を出した場合の警告です
   - CLIPPED の場合、表示されている予測上昇率は ±30% にキャップされています
   - 実際の投資判断の際は信頼区間も合わせて参照してください

7. マクロバナーが表示されない
   - `FRED_API_KEY` が設定されていない場合、バナーは表示されません
   - GitHub Secrets に `FRED_API_KEY` を追加してください（無料取得可）

8. ポジションサイジングが表示されない
   - 株価データ（年率ボラティリティ）の取得に失敗した場合はスキップされます

9. バックテスト品質が「試行数不足」になる
   - `config.yaml` の `backtest.num_rules_tested` が `min_rules_for_pbo` 未満の状態です
   - 現在の実装では 1 戦略のみのため、pbo / Reality Check は算出不可です

## 6. 注意点

- 予測は将来の利益を保証しません
- 小額での検証運用から開始し、ルールを固定して評価してください
- 取引前に手数料・税金・為替コストを必ず考慮してください
- ポジションサイジングの値は機械的な推定です。実際のリスク許容度に応じて調整してください
- バックテスト品質指標（PBO・Reality Check）は簡略推定値です。過信しないでください
