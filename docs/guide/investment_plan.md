# 運用計画ガイド

このシステムを使って実際に投資するうえでの現実的な見通しと運用方法をまとめます。

---

## 1. このシステムで儲けられるか？

### 正直な前提

- 本システムは **投資判断の補助ツール** です。自動売買は行いません。
- AI予測が必ず利益をもたらす保証はありません。
- 的中率50%超＋損切り規律の組み合わせで初めて利益が期待できます。

### 短期（〜1年）

| 期間 | 現実的な見通し |
|------|--------------|
| 0〜3ヶ月 | データ蓄積フェーズ。実際の運用開始は推奨しない |
| 3〜6ヶ月 | 的中率データが揃ってくる。シミュレータで仮検証するタイミング |
| 6〜12ヶ月 | 実績ベースの評価が可能になる。累計的中率 55〜60%+ が投資開始の目安 |

**短期で確実に儲ける方法はありません。** モデルが市場を継続的に上回れるかどうか自体、最低1〜2年分のデータがないと評価できません。

### 長期（1〜3年）

- 年率リターンの評価には **最低1年以上の運用実績** が必要
- 戦略比較グラフ（`accuracy.html`）でSPYベンチマーク（市場平均）と比較しながら評価
- **市場平均（年率7〜10%）を上回れば優秀**。それ以下なら個別銘柄より投資信託（インデックス）の方が合理的

## 2. 初期投資の目安

### 米国株（本システムの対象）

現在の設定では **S&P500・NASDAQ100** の銘柄を対象にしています（`config.yaml: markets`）。

| 投資フェーズ | 金額目安 | 説明 |
|------------|---------|------|
| 検証フェーズ | $500〜$1,000 | 少額でシステムの動作確認。損失許容範囲内で開始 |
| 本格運用（小）| $3,000〜$5,000 | 上位10銘柄に分散。1銘柄あたり $300〜$500 |
| 本格運用（中）| $10,000〜 | 推奨ウェイト（`stock.html`に表示）を活用した本格分散 |

**ポジションサイジング参考（`stock.html`の「ポジションサイジング」パネル）:**
- 目標年率ボラティリティ10%基準で推奨ウェイトが自動計算されます
- 例: 総資金 $10,000 × 推奨ウェイト15% = 1銘柄あたり $1,500

### 日本株

現在の設定では日本株（日経225）は**無効**です（config.yamlでコメントアウト）。

```yaml
# - "nikkei225"  # 必要に応じて有効化
```

日本株を追加したい場合は `config.yaml` の該当行を有効化してください。
日本株は最低売買単位が100株のため、銘柄によって最低投資額が大きく異なります。

| 銘柄価格帯 | 1単元の目安 |
|-----------|-----------|
| 500円台 | 5万円程度 |
| 3,000円台 | 30万円程度 |
| 10,000円台 | 100万円程度 |

日本株で分散投資（5〜10銘柄）するなら **最低でも50〜100万円** が現実的です。

## 3. データ蓄積3ヶ月後の運用ステップ

### Step 1: シミュレータで仮検証（3ヶ月後）

`simulator.html` を使い、実際の投資をする前に **過去の予測データでシミュレーション** します。

1. 「シミュレーション実行」で仮想損益を確認
2. **累計損益がプラス** かつ **市場平均（SPY）を上回っている** 場合に次のステップへ
3. マイナスまたはSPYを下回る場合は、もう3ヶ月データを蓄積して再評価

### Step 2: 的中率の確認基準（`accuracy.html`）

| 指標 | 投資開始の目安 |
|------|-------------|
| 累計的中率 | **55%以上**（50%は無意味、コインフリップと同じ） |
| 週次的中率の安定性 | 週ごとのバラつきが小さい（グラフで確認） |
| 予測誤差（MAE） | 小さいほど良い（誤差分析パネルで確認） |

### Step 3: 少額から開始（6ヶ月後）

- まず **全体資金の5〜10%** だけ使って実投資を開始
- 毎週のSlack通知を確認し、上位銘柄の1〜2銘柄だけに絞る
- 損切りめやす（`stock.html`に表示）を必ず設定する

### Step 4: 週次レビューと月次評価

**毎週やること:**
1. Slack通知を確認（銘柄・予測価格・信頼区間）
2. 証券会社画面で実際の価格を再確認してから判断
3. 損切りラインを超えていないかチェック

**毎月やること:**
1. `accuracy.html` で累計的中率と戦略比較を確認
2. SPYベンチマークと比較し、劣後している場合は投資額を増やさない
3. `VIX > 25` または 逆イールド時はポジションを縮小（マクロバナーで確認）

## 3-A. 具体的な資金計画（社会人2年目・手取り月21万・貯金180万の場合）

### 緊急予備資金（絶対に確保する）

```
月の生活費 ≈ 手取り21万 − 貯金5万 = 16万円
緊急予備資金 = 16万 × 6ヶ月 = 96万円
```

| 資金区分 | 金額 | 動かせるか |
|---------|------|----------|
| 緊急予備資金 | **96万円** | 絶対に触らない |
| 今すぐ投資可能 | **84万円** | 180万 − 96万 |
| 実際に使う初期投資 | **15〜30万円** | 投資可能額の1/4以下から開始 |

### 具体的な投資スケジュール

| 時期 | 行動 | 金額 |
|------|------|------|
| 今〜3ヶ月後 | 何もしない（データ蓄積のみ） | 投資ゼロ |
| 3ヶ月後 | シミュレータで仮検証 | 投資ゼロ |
| 6ヶ月後 | 的中率55%以上なら実投資開始 | $1,000〜$2,000（約15〜30万円） |
| 毎月 | 貯金5万のうち2万を積み立て | $100〜$150/月 |
| 1年後 | 戦略比較でSPY超えを確認したら増額 | 状況次第で追加 |

### なくなっても大丈夫な金額の目安

- 初期: **15万円**（万が一ゼロになっても生活に影響なし）
- 毎月: **2万円**（月の貯金5万から）
- 1年で最大 **39万円** の投資額になる計算

## 4. リスク管理の要点

### 損切り規律

`stock.html` の「ポジションサイジング」パネルに表示される **損切りめやす** を必ず守る。

例: `-6.9%` → 購入価格から6.9%下落したら迷わず売却

### 分散の原則

- 1銘柄に全資金の20%以上は入れない（`config.yaml: max_weight_cap: 0.20` の設定値）
- リスクオフ時（VIX高・逆イールド）は全体のポジションを下げる

### 税金・コスト

| コスト種別 | 米国株の場合 |
|-----------|-----------|
| 売買手数料 | ネット証券なら無料〜0.1%程度 |
| 為替コスト | 円→ドル換算で往復0.5〜1%程度 |
| 税金 | 利益の20.315%（確定申告または特定口座） |
| 配当課税 | 米国源泉10% + 国内10.21% = 約20.315% |

## 5. 現実的な期待値

| シナリオ | 年率リターン目安 | 可能性 |
|---------|---------------|-------|
| 最悪ケース | -20〜-30%（市場暴落時） | 低いが起こりうる |
| 並み | 0〜5%（市場平均以下） | 普通にありうる |
| 良いケース | 10〜20%（市場平均程度〜やや上回る） | 目指す水準 |
| 最良ケース | 20%超 | システムが継続的に機能している場合 |

> **重要:** このシステムで継続的に年率20%超を狙うのは現実的ではありません。
> 年率10〜15%を安定的に維持できれば、プロのファンドマネージャーと同等です。

## 6. まとめ：推奨タイムライン

```
[今] データ蓄積開始（システム稼働）
 ↓
[3ヶ月後] シミュレータで仮検証 → 累計的中率55%未満なら継続観察
 ↓
[6ヶ月後] 少額（全体の5〜10%）で実投資開始
 ↓
[1年後] 戦略比較グラフでSPYと比較 → 上回っていれば増額、下回れば停止検討
 ↓
[2〜3年後] 安定した実績が確認できれば本格運用
```

**最初の3ヶ月は何もしない（データを貯める）のが正しい選択です。**
