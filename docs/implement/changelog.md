# ルール変更ログ

> 目的: ウォークフォワード評価の「ルール固定期間」を守るため、
> スクリーニング条件・モデル設定・重みの変更履歴をここに記録する。
>
> **変更タイミング:** 評価期間（test_weeks）中は変更禁止。
> 変更は「次のトレイン窓の開始時」のみ許可。
> 変更後は必ずここに記録してから config を適用すること。

---

## テンプレート

```
## YYYY-MM-DD｜変更者: <名前>

### 変更内容
- config.yaml の `<キー名>` を `<旧値>` → `<新値>` に変更

### 変更理由
- <変更を決めた根拠・観測された問題など>

### 影響範囲
- スクリーニング / 予測 / ウォークフォワード評価 のどれに影響するか

### 次のトレイン窓
- 開始予定: YYYY-MM-DD（この日以降のデータからルール変更が有効）
```
